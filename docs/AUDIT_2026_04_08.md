## АРХИТЕКТУРНЫЕ ПЛЮСЫ

1. **Kafka** — at-least-once delivery, manual commit, idempotency guard
2. **Rate Limiter** — атомарный Lua-скрипт, 2 уровня (global + per-chat)
3. **Session Pool** — round-robin, anti-detect jitter, FloodWait cooldown, auto-ban
4. **Telethon Factory** — полный device fingerprint, X-Pay JSON support
5. **Bot Pool** — lazy load, platform binding, fallback, auto-deactivate
6. **Security** — Fernet encryption для токенов и сессий
7. **Observability** — Prometheus + structlog + Sentry
8. **Tests** — 8 файлов, handlers/router/schemas/idempotency/security
9. **CI/CD** — GitHub Actions + multi-stage Docker

## АРХИТЕКТУРНЫЕ ПРОБЛЕМЫ

| # | Проблема | Severity |
|---|----------|----------|
| 1 | Sequential consumer (bottleneck при нагрузке) | ⚠️ Medium |
| 2 | Нет retry для скрапинга (1 попытка) | ⚠️ Medium |
| 3 | ProxyManager: TCP-only проверка | ⚠️ Low |
| 4 | Нет Docker healthcheck | ⚠️ Low |
| 5 | men_percent всегда 0 (API limitation) | ℹ️ Info |
| 6 | Backend producer — новое TCP на каждое сообщение | 🔴 High (в бекенде) |

## НАГРУЗКА

- **Posting**: 25 msg/sec → 1000 постингов за ~40 сек (happy path) / ~2 часа (worst case с retries)
- **Scraping**: 4 аккаунта → ~100-200 скрапингов/час. 1000 площадок = ~42 мин (happy path)
- **Масштабирование**: Kafka consumer group + shared Redis rate limiter = горизонтальное масштабирование возможно
